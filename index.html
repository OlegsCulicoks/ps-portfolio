List of projects

Memory Game

Lietotāji var reģistrēties, pieteikties, un rediģēt savus profilus. Punkti tiek aprēķināti pēc veiksmīgi atrastajiem pāriem, un ir pieejams top saraksts. Saglabā spēlētāju vēsturi un pabeigto līmeņu rezultātus. Lietotāji var mainīt fonu un baudīt animācijas spēles laikā. Līmeņi kļūst arvien sarežģītāki, ar kopumā 20 līmeņiem un laika ierobežojumiem.

Game / Spēle

Lietotāji var izveidot jaunu kontu vai pieteikties ar esošo. Spēlei ir jābūt pamata funkcijām, kā spēlētāja kustības un šķēršļu ģenerēšana, kā arī grafiskais dizains ar vizuāliem elementiem. Spēles mehānika ietver punktu uzskaiti, dzīvības un uzlabojumus, kas iegādājami ar Stripe. Lietotāja saskarnei jāietver izvēlnes, profila rediģēšana, vēsture, līderu saraksts un veikals. Spēle būs ar skaņas efektiem, mūziku, optimizēta un pārbaudīta, lai novērstu kļūdas.

User Roles / Lietotāju lomas

Administrators pārvalda lietotāju kontus, piešķir piekļuves tiesības, un ģenerē atskaites par noliktavas darbībām. Viņš arī rediģē, dzēš un pievieno datus sistēmā. Noliktavas darbinieks ievada un pārvalda preču informāciju, veic pasūtījumu izveidi, pieņemšanu un izpildi, un ģenerē atskaites par noliktavas darbību. Plauktu kārtotājs organizē preces uz plauktiem, sagatavo preces atskaitēm, un ievada informāciju par preču stāvokli sistēmā. Katram lietotājam ir specifiskas atbildības, lai nodrošinātu efektīvu noliktavas pārvaldību.

Learn More

Ticket Purchase Site / Biļešu Iegādes Vietne

Lietotāji var reģistrēties un pierakstīties drošā autentifikācijas sistēmā. Administratori var pievienot pasākumus ar nosaukumu, datumu, laiku, biļešu veidiem, vietu un cenām. Lietotāji var pārskatīt pieejamās biļetes un iegādāties tās, ar maksāšanas sistēmu pieslēgtu. Pasākumi tiek parādīti dinamiskā sarakstā ar filtrēšanas un meklēšanas iespējām. Lietotāji var atstāt atsauksmes par biļetēm un pasākumiem. Pēc pirkuma, lietotāji var saglabāt biļeti PDF formātā ar QR kodu. Pirkumu vēsture ļauj lietotājiem pārskatīt iepriekšējos pirkumus un saņemt apstiprinājumus.

Learn More

Group Work / Grupas Darbs

Izveido tīmekļa vietni ar lietotāja reģistrācijas un pieteikšanās sistēmu. Lietotāji var pievienot uzdevumus ar nosaukumu, aprakstu un termiņu, kā arī apskatīt savus uzdevumus sarakstā. Uzdevumus var atzīmēt kā pabeigtus vai nepabeigtus un dzēst tos. Iekļauj darba plānošanas kalendāru uzdevumu termiņu pārskatīšanai un meklēšanas funkciju uzdevumu atrašanai. Datu bāzē glabā lietotāju reģistrācijas datus un uzdevumu informāciju, ar iespēju atgūt paroli. Tīmekļa vietnei jābūt responsīvai un pielāgojamai dažādām ierīcēm. Papildus funkcionalitāte var ietvert projektu vai kategoriju izveidi, uzdevumu prioritizāciju un atgādinājumus.

Learn More

Task Management System / Uzdevumu Pārvaldības Sistēma

Izveido MySQL datu bāzi "task_management" ar tabulu "tasks", kurai ir kolonnas: id, title, description, due_date, status, created_at. Izstrādā PHP OOP skriptu, lai savienotu ar MySQL un īstenotu API punktus uzdevumu pārvaldībai (GET, POST, PUT, DELETE). Validē ievades datus un pievieno kļūdu apstrādi ar JSON ziņojumiem. Izveido lapošanu uzdevumu sarakstam ar lietotāja izvēlētu lapas numuru un uzdevumu skaitu. Pārbaudi API ar HTML/JavaScript saskarni, pārbaudot izveides, lasīšanas, atjaunināšanas un dzēšanas darbības. Implementē pamata autentifikāciju, lai aizsargātu API punktus. Bonusa uzdevums: pievieno uzdevumu kārtošanas, filtrēšanas iespējas un lietotāju autentifikāciju ar piekļuves ierobežojumiem.
